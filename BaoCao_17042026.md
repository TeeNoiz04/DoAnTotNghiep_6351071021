**Framework:** ABP Framework (abp.io) — ASP.NET Core  

---

## 1. Tổng quan ABP Framework

**ABP Framework** (abp.io) là một nền tảng phát triển ứng dụng web mã nguồn mở, xây dựng trên ASP.NET Core. ABP cung cấp sẵn kiến trúc theo **Domain-Driven Design (DDD)**, hệ thống module hóa, phân quyền, multi-tenancy, audit logging và nhiều tính năng enterprise khác.

### Các đặc điểm nổi bật

- Kiến trúc **modular** — mỗi tính năng là một module độc lập, có thể tái sử dụng
- Tích hợp sẵn **Identity Server**, **OpenIddict** cho xác thực & phân quyền
- Hỗ trợ **Multi-Tenancy** out-of-the-box
- **Domain Events**, **Application Services**, **Repositories** theo chuẩn DDD
- Auto API Controllers — tự động expose Application Service thành REST API
- Hỗ trợ nhiều UI: Blazor, MVC Razor Pages, Angular, React

```csharp
    : CrudAppService<PriceOffer, PriceOfferDto, Guid,
                     GetPriceOfferListInput, CreateUpdatePriceOfferDto>,

        entity.Approve(CurrentUser.UserName, note);
        await Repository.UpdateAsync(entity);

    {
        var entity = await Repository.GetAsync(id);
        entity.Reject(reason);
        await Repository.UpdateAsync(entity);
```

### 3.6 HttpApi — Tầng API Controllers

**Phụ thuộc:** `Application.Contracts`

Trong ABP, hầu hết các controller được **tự động tạo** từ `IApplicationService` qua **Auto API Controllers** — không cần viết controller thủ công. Tuy nhiên vẫn có thể override:

```
QuoteFlow.HttpApi/
├── QuoteFlowHttpApiModule.cs
├── Controllers/
│   └── QuoteFlowController.cs      ← Base controller (nếu cần override)
└── PriceOffers/
    └── PriceOfferController.cs     ← Custom endpoints (upload, export...)
```

**Auto API — ABP tự expose:**
```
GET    /api/app/price-offer              → GetListAsync
GET    /api/app/price-offer/{id}         → GetAsync
POST   /api/app/price-offer              → CreateAsync
PUT    /api/app/price-offer/{id}         → UpdateAsync
DELETE /api/app/price-offer/{id}         → DeleteAsync
POST   /api/app/price-offer/{id}/submit  → SubmitAsync
POST   /api/app/price-offer/{id}/approve → ApproveAsync
POST   /api/app/price-offer/{id}/reject  → RejectAsync
```

**Custom Controller (ví dụ export Excel):**
```csharp
[Route("api/quote-flow/price-offer")]
public class PriceOfferController : QuoteFlowController
{
    [HttpGet("export-rejected")]

}
```

## 7. Luồng xử lý một request điển hình

Lấy ví dụ: **Sale phê duyệt một Price Offer**

```
[Blazor UI]
  └─> Click "Approve" button
        └─> Gọi IPriceOfferAppService.ApproveAsync(id, note)
              └─> [HttpApi - Auto Controller]
                    POST /api/app/price-offer/{id}/approve
                          └─> [Application Layer - PriceOfferAppService]
                                1. Check permission: QuoteFlow.PriceOffers.Approve
                                2. Load entity: await Repository.GetAsync(id)
                                3. Gọi domain method: entity.Approve(currentUser, note)
                                   └─> [Domain Layer - PriceOffer.cs]
                                         - Validate state machine (phải là InProgress)
                                         - Cập nhật ApprovalStatus
                                         - Tạo ApprovalHistory record
                                         - Raise Domain Event: PriceOfferApprovedEvent
                                4. await Repository.UpdateAsync(entity)
                                   └─> [EF Core Layer]
                                         - Persist to AppPriceOffer
                                         - Persist to AppApprovalHistories
                                5. Return PriceOfferDto (AutoMapper)
```

## 8. Điểm lưu ý khi phát triển

**Không đặt business logic trong AppService** — toàn bộ rule (validate trạng thái, tính toán, phát event) phải nằm trong Domain Entity hoặc Domain Service.

**Repository chỉ query, không chứa business logic** — tránh đặt điều kiện nghiệp vụ trong `IQueryable` filter của Repository.

**DTO không được reference Entity trực tiếp** — Application Layer dùng AutoMapper để convert, tránh expose Domain object ra ngoài.

**Permission phải được khai báo trong `PermissionDefinitionProvider`** trước khi dùng `[Authorize(...)]` — nếu thiếu bước này, permission sẽ không hiển thị trong trang quản lý phân quyền.

**Migration phải chạy qua `DbMigrator`** (project riêng) — không chạy `dotnet ef database update` trực tiếp trên Web project trong môi trường production.
